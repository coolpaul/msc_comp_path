2.2 Operating Systems

2.2.1 Windows

2.2.1.1 Windows Subsystem for Linux

You can run Linux on a Windows machine using Windows Subsystem for Linux, using Windows and Linux side by side. <https://learn.microsoft.com/en-us/windows/wsl/install.

2.2.2 Linux

Can run on the same hardware as Windows machines, but the operating system is different and based on Unix.The Unix philosophy has small specialised programs that do one thing extremely well and can be linked together using ‘pipes’ to perform mor complex task. There is a hierarchical file system that treats devices as files, making the environment more powerful.

2.2.3 Apple (OSX)

Apple has a Unix operating system, similar to Linux.

2.2.3.1 Intel

The Intel processor (x86-64 architecture ) is used on older machines.

2.2.3.2 Apple Silicon (ARM; M processors)

More recently, Apple developed its own M series (M1 - M5) using RISC architecture focusing on efficiency, power life and high-performance per watt. The Apple Silicon chips integrate the CPU, GPU and RAM together for faster communication (whilst these are separate on Intel systems). Intel based applications can only run on a Apple processor via Rosetta 2 translation.

2.3 Scanners

Several scanners are available to scan histology slides. Need to name different ones. Files can be very large and this is dependent on the amount of tissue on the slide. Can easily be 3 - 4 GByte.

2.3.1 Pyramid file structure

For example the 3DHistech scanners create mrxs files. An example is included. The advantage is that not the whole slide will need to be downloaded to see a particular patch, but zoom in to what you want to see (ie. google maps). This is particularly useful for diagnostics.

2.3.2 Singe file

Like TIFF and SVS. An example is included. The whole file will need to be downloaded for viewing. The file structure is simpler and this file structure is more suited for machine learning / artificial intelligence where all pixels are required.

2.3.3 DICOM

Standard, but not universal

2.5 Excercise

Open the svs file that is included in the course in Qupath. Select a region of interest (ROI) and annotate some osteoclast like giant cells. Export this region of interest as a geojson file. Import the svs file and geojson file in python and show the region of interest. Save this file as a jpg. Also, using the [create_patches.py]{https://github.com/coolpaul/msc_comp_path/tree/main/code} script you have downloaded to your computer, divide the file into patches. Bonus: stich the file patches back together again.
